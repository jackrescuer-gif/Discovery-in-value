# Процесс Discovery инициатив - Навигация

## 📁 Структура документации

### Основные файлы:

1. **README.md** - Полное описание процесса Discovery
   - Обзор процесса
   - Этапы процесса
   - Входные и выходные данные
   - Критерии готовности
   - Роли и ответственность
   - Метрики успеха

2. **process-diagram.md** - Блок-схемы процесса
   - Основной процесс Discovery (Mermaid)
   - Детальный процесс валидации гипотез
   - Процесс прототипирования и тестирования
   - Временная шкала (Gantt chart)
   - Роли и взаимодействие

3. **executive-dashboard.html** - Руководительский дашборд (C-level)
   - Дашборд в стиле McKinsey для CEO/CPO/CTO/CFO/COO
   - KPI: затраты, количество исследований, подтверждения, скорость исследований и проверки гипотез
   - Финансовый, операционный и стратегический блоки с action title
   - Фильтры по периоду/направлению/команде/статусу и рекомендации по рискам

4. **executive-dashboard-data-dictionary.md** - Словарь данных и формул KPI
   - Определения метрик и бизнес-правила расчёта
   - Формулы TVC, Success Rate, Velocity, Cost per Validated
   - Data quality и пороги алертов

## 🚀 Как использовать

### Просмотр сайта:

1. Откройте файл `index.html` в браузере — портал по потоку создания ценности
2. Переходите по ссылкам на нужные страницы:
   - **Этапы процесса** — value-stream.html
   - **Руководительский дашборд (C-level)** — executive-dashboard.html
   - **Выбор фреймворка** — framework-selection.html

### Просмотр блок-схем в Markdown:

Откройте файл `process-diagram.md` в любом редакторе с поддержкой Mermaid:
- GitHub
- GitLab
- VS Code с расширением Mermaid
- Notion
- Confluence

### Чтение документации:

Откройте `README.md` для полного описания процесса.

## 🎯 Быстрый старт

### Для Product Manager:
1. Прочитайте раздел "Роли и ответственность" в README.md
2. Изучите основной процесс в value-stream.html и process-board.html
3. Используйте временную шкалу для планирования

### Для UX Researcher:
1. Изучите раздел "Валидация гипотез" в value-stream.html и README.md
2. Ознакомьтесь с методами валидации в README.md
3. Используйте процесс валидации как чеклист

### Для UX/UI Designer:
1. Изучите раздел "Прототипирование" в value-stream.html
2. Ознакомьтесь с инструментами прототипирования
3. Используйте процесс как руководство для работы

### Для Tech Lead:
1. Изучите этап "Техническое прототипирование"
2. Ознакомьтесь с критериями готовности к production
3. Используйте процесс для оценки сложности
